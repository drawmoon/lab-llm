
### RRF Rerank

RRF（Reciprocal Rank Fusion） 是一种 无模型、无训练的排序融合算法。

>专业项目中常将 **RRF 作为召回融合层** → 再用 **Cross-Encoder / LLM Reranker** 做精排。




```python
from collections import defaultdict


def rrf_rerank(
    documnents: list[list[str]], k: int = 60, limit: int | None = None
) -> list[tuple[str, float]]:
    """
    Reciprocal Rank Fusion (RRF)

    Args:
        documnents: 多个有序结果列表
        k: 平滑参数（通常 50~100）
        limit: 返回 top-N（None 表示全量）

    Returns:
        [(doc_id, rrf_score), ...] 按 score 降序
    """
    scores = defaultdict(float)

    for ranked in documnents:
        for rank, doc_id in enumerate(ranked, start=1):
            scores[doc_id] += 1.0 / (k + rank)

    ranked = sorted(scores.items(), key=lambda x: x[1], reverse=True)

    return ranked[:limit] if limit else ranked
```


**示例代码**

```python
bm25_ranked = ["docA", "docB", "docC", "docD"]
bge_embed_ranked = ["docB", "docE", "docA", "docF"]
qwen_embed_ranked = ["docC", "docA", "docG", "docH"]

data = [bm25_ranked, bge_embed_ranked, qwen_embed_ranked]

# ============================================
# 使用默认的 k=60
# ============================================
r = rrf_rerank(data)
print("RRF result (default k=60):")
for doc_id, score in r:
    print(f"  Doc ID: {doc_id}, Score: {score:.4f}")

print("\n")

# ============================================
# 指定 k 值和 limit
# ============================================
r2 = rrf_rerank(data, k=20, limit=3)
print("RRF result (k=20, limit=3):")
for doc_id, score in r2:
    print(f"  Doc ID: {doc_id}, Score: {score:.4f}")
```

## 二、RRF 在 Milvus 中的原理

### 📌 核心思想

RRF 偏重 **“不同检索路径的排名一致性”**：

- 两个或多个搜索路径各自返回排名列表
    
- 不直接用 score，而是基于每条结果的 **rank 排名位置进行融合**
    
- 最终 score = ∑ (1 / (k + rank_i(d)))  
    → **越靠前的 rank 累计得分越高**。([Milvus](https://milvus.io/docs/id/rrf-ranker.md?utm_source=chatgpt.com "RRF Ranker | Milvus Documentation"))
    

这里的 `k` 是平滑参数（默认 60，可调），用来控制前 n 名项的贡献分布。([Milvus](https://milvus.io/docs/id/rrf-ranker.md?utm_source=chatgpt.com "RRF Ranker | Milvus Documentation"))

### 📌 举例说明

假设两个路径的 Top-5:

|id|sparse rank|dense rank|
|---|---|---|
|101|1|2|
|198|4|1|
|175|5|4|
|203|2|—|
|150|3|—|

RRF 融合后按综合得分排序，交叉出现的更高一致性者胜出。([Milvus](https://milvus.io/docs/id/rrf-ranker.md?utm_source=chatgpt.com "RRF Ranker | Milvus Documentation"))

## 一、Cohere Rerank 是什么？

### 1️⃣ 定义

**Cohere Rerank** 是一种 **基于大模型（Cross-Encoder）** 的重排序（Re-ranking）服务。  
它对「**查询 + 候选文档**」进行**逐对语义理解打分**，然后重新排序。

> 核心思想：  
> 👉 _“不是看向量距离，而是让模型真正读懂 query 和 document 是否相关”_

---

### 2️⃣ 工作原理

1. 先用 **BM25 / Embedding / 混合检索**召回 Top-N（如 50~200 条）
    
2. 将 `(query, document)` 送入 Rerank 模型
    
3. 模型输出 relevance score
    
4. 按 score 排序，取 Top-K
    

📌 本质是 **Cross-Encoder**：

- 输入：`[CLS] query [SEP] document`
    
- 输出：相关度分数
    

---

### 3️⃣ 优点

✅ **语义理解能力极强**  
✅ 对复杂、长 Query 表现好  
✅ 能纠正 embedding / BM25 的误召回  
✅ 对自然语言、问句、业务表达非常友好

### 4️⃣ 缺点

❌ 成本高（token + API）  
❌ 延迟高（O(N) 推理）  
❌ 不适合直接全量检索，只能 rerank 小集合

---

### 5️⃣ 典型应用场景

- **RAG 最终结果排序**
    
- **BI 自然语言查询**
    
    - 如：`“中天钢铁今年售电量”`
        
- **企业知识库 / FAQ**
    
- **搜索质量兜底层**
    
- **多路召回后的精排层**
    

---

## 二、RRF Rerank 是什么？

### 1️⃣ 定义

**RRF（Reciprocal Rank Fusion）** 是一种 **无模型、无训练的排序融合算法**。

> 核心思想：  
> 👉 _“一个结果在多个排序中都靠前，就更重要”_

---

### 2️⃣ 算法公式

[  
Score(d) = \sum_{i=1}^{n} \frac{1}{k + rank_i(d)}  
]

- `rank_i(d)`：文档 d 在第 i 个检索器中的排名
    
- `k`：平滑参数（通常 60）
    

---

### 3️⃣ 工作方式

1. 使用多个检索器：
    
    - BM25
        
    - Embedding
        
    - 规则检索
        
2. 分别得到 Top-N 列表
    
3. 使用 RRF 融合排序
    
4. 输出统一排序结果
    

📌 **不看分数，只看排名**

---

### 4️⃣ 优点

✅ 极其简单  
✅ 无模型、无成本  
✅ 对不同检索器鲁棒  
✅ 延迟极低  
✅ 工程实现非常友好

### 5️⃣ 缺点

❌ 没有语义理解  
❌ 无法处理复杂逻辑关系  
❌ 排名只依赖“是否靠前”，无法精细区分

---

### 6️⃣ 典型应用场景

- **BM25 + Embedding 混合检索**
    
- **召回层融合**
    
- **成本敏感系统**
    
- **BI 场景的第一层排序**
    
- **大规模检索系统（如 Elasticsearch + 向量）**
